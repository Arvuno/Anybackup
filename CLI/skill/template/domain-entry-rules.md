# Domain Entry Skill Rules

本文件定义 Foundation CLI 各业务域入口型 skill 的通用规则。

适用对象：

- `skill/<domain>/SKILL.md`
- `skill/<domain>/references/commands.md`
- `skill/<domain>/references/commands/*.md`

目标不是写出“内容很多”的技能，而是写出“agent 能快速做对事”的技能。

## 一、入口 skill 的职责边界

入口型 `SKILL.md` 只负责：

1. 判断当前用户意图是否属于本业务域。
2. 把用户意图映射到正确命令。
3. 引导 agent 打开正确的单命令文档。

入口型 `SKILL.md` 不负责：

1. 承载完整请求体规范。
2. 承载大段示例命令集合。
3. 承载完整排错手册。
4. 承载所有返回字段明细。

这些细节必须下沉到 `references/commands/*.md`。

## 二、推荐阅读路径

每个业务域 skill 都应遵循同一条路径：

`SKILL.md -> references/commands.md -> references/commands/*.md`

要求：

1. `SKILL.md` 是入口页。
2. `references/commands.md` 是索引页。
3. `references/commands/*.md` 是单命令详情页。
4. 不要让 agent 在多个公共 reference 页面之间来回跳转。

## 三、SKILL.md 必备结构

每个入口型 `SKILL.md` 至少包含以下区块：

1. 适用场景
2. 不适用场景
3. 标准流程
4. 快速判断规则
5. 写入与只读
6. 必要的全局上下文
7. 常见误判
8. 意图到命令
9. 参考资料

## 四、SKILL.md 必须回答的问题

agent 打开入口 skill 后，必须能立即回答：

1. 这是不是当前 domain 的问题？
2. 这是写操作还是只读操作？
3. 最容易误选的命令是哪几组？
4. 需要先拿哪个上游 ID？
5. 下一步该打开哪个文件？

如果这 5 个问题不能在入口页里快速回答，说明入口 skill 还不够好。

## 五、快速判断规则的写法

不要只写“写入 vs 只读”的粗分类。
必须补充能阻止误选命令的判定规则。

常见有效写法：

- “如果用户要历史记录，用 A；如果用户要执行日志，用 B”
- “如果用户要实例列表，用 A；如果用户要类型字典，用 B”
- “如果命令依赖上游上下文，明确写出必须提供的 ID”

入口 skill 应优先列出最容易混淆的命令对，而不是平均描述所有命令。

## 六、索引页 commands.md 的职责

`references/commands.md` 只负责命令选择，不负责完整说明。

必须包含：

1. 读写分组
2. 意图到命令映射
3. 最容易漏掉的参数
4. 单命令文档跳转

不应包含：

1. 大段示例
2. 公共排错手册
3. 长篇请求体说明
4. 重复的返回结构细节

## 七、单命令文档的推荐结构

每个 `references/commands/*.md` 建议保持统一结构：

1. 命令概览
2. 何时使用
3. 必填上下文
4. 参数要点或请求体要点
5. 最小可执行示例
6. 常见错误
7. 相关跳转

如果命令复杂，再追加：

1. 返回结果
2. 枚举类型
3. 返回案例

## 八、公共 reference 的收敛原则

如果存在 `examples.md`、`error-playbook.md`、`request-body-schema.md` 这类公共页，应优先判断它们是否真的必要。

默认原则：

1. 示例应优先下沉到对应单命令文档。
2. 请求体要点应优先下沉到写命令文档。
3. 排错要点应优先写成命令级“常见错误”。
4. 只有当多个命令共享大量不可拆分规则时，才保留公共页。

## 九、入口 skill 的常见反模式

以下情况说明 skill 需要重构：

1. `SKILL.md` 同时承担入口、示例、排错、schema 四种职责。
2. 删除文件后，入口页仍保留失效引用。
3. agent 必须在两个以上公共 reference 之间来回切换，才能拼出一个命令。
4. 入口页只有“写入 vs 只读”，没有误判防护。
5. 单命令文档缺失，导致索引页有入口但没有落点。

## 十、自检清单

在新增或修改某个 domain skill 后，至少检查：

- `SKILL.md` 是否只承担入口职责
- `commands.md` 是否只承担索引职责
- 所有链接是否都能落到存在的文件
- 最易混淆命令是否已经前置说明
- 所有依赖上游 ID 的命令是否已明确写出
- 写命令是否明确要求 `--data` 或其他关键参数
- 删除公共页后是否仍有残留引用

# AG-UI 可见性策略

## 禁止展示

- 业务知识网络名称、知识网络 ID、索引名、内部库表名。
- 对象类 ID、关系类 ID、动作类 ID、租户内部 ID、运行时 trace。
- 函数式工具名、方法名、类名、模块名和内部 API 路径。
- 原始工具参数、原始工具返回、未脱敏日志、异常堆栈、连接串、密钥、令牌。
- 系统提示词、开发者提示词、完整内部推理链。

## 语义名称映射

| 内部能力 | 用户可见语义名称 |
| --- | --- |
| 知识检索能力 | 恢复知识检索 |
| 知识结构分析能力 | 恢复知识结构分析 |
| 对象查询能力 | 恢复对象查询 |
| 动作准备能力 | 恢复动作准备 |
| 恢复点检索能力 | 恢复点查询 |
| 可用性校验能力 | 恢复可用性验证 |

不得把内部能力名原样放入用户可见文案。需要描述检查范围时，使用“恢复经验检查范围”“恢复运行数据检查范围”“保护对象检查范围”等语义化表达。

## 脱敏表达

- 对象：显示业务对象类型和脱敏名称，例如“目标 MySQL 实例”。
- 时间：显示业务时间范围，例如“最近一次备份时间点”。
- 参数：显示筛选意图，例如“按最新备份时间点范围查询”。
- 返回：显示命中数量、结论和下一步，例如“已找到 3 个可用时间点”。

## 显示目标
在没有命中 MySQL 可用恢复点时，让用户看到检查范围、空结果依据和下一步。

## 内容约束
允许展示目标对象类型、脱敏实例名称、检查时间范围、检查项和空结果原因。不得展示内部检索源或底层返回。

## 布局约束
根布局使用 `stack`。使用 `callout` 表达空结果结论，`kv-list` 展示检查范围，`data-table` 展示检查项和状态，`action-row` 展示重试或补充条件。

## 思考链展示
展示“已完成最新时间点检查，当前没有命中可用备份”，下一步为等待用户调整范围或重试。

## 工具调用展示
业务能力名称显示为“恢复点查询”。状态从“执行中”更新为“已完成”，结果摘要显示“未命中可用时间点”。

## 结果返回展示
标题为“未找到可用恢复点”。摘要说明已检查对象、时间范围、可能原因和可执行下一步。

## 禁止展示
不得展示内部检索源名称、内部能力名、对象类标识、原始参数、原始返回、连接串、日志或堆栈。

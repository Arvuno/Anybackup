## 显示目标
展示 MySQL 恢复点查询的工具调用进度，使用户知道正在检查什么。

## 内容约束
展示业务能力、状态、目的、脱敏检查范围和当前进展。只展示业务语义，不展示内部方法名。

## 布局约束
根布局使用 `stack`。工具调用块包含 `heading`、`kv-list`、`data-table` 和 `callout`。

## 思考链展示
展示“正在获取恢复点候选信息”，下一步为汇总命中时间点。

## 工具调用展示
业务能力名称显示为“恢复点查询”。表格行使用“保护对象”“备份时间点视图”“恢复数据检查范围”等语义化检查项。

## 结果返回展示
工具完成后更新同一槽位，展示命中数量、最新时间点或空结果依据，并保持后续结果独立输出。

## 禁止展示
不得展示内部工具名、内部检索源、对象类标识、原始参数、原始返回、连接串、日志或堆栈。

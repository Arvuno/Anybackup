## 显示目标
展示多个 MySQL 恢复候选方案，支持用户比较并选择。

## 内容约束
每个候选只展示脱敏恢复点时间、目标对象、可用性、风险摘要、推荐理由和动作。不得展示内部对象标识。

## 布局约束
根布局使用 `stack` + `grid`。每个候选使用 `card`，卡片内使用 `metric-list`、`kv-list` 和 `badge-row`。底部使用 `action-row`。

## 思考链展示
展示“已获得候选恢复点，正在整理可比较方案”，下一步为等待用户确认。

## 工具调用展示
业务能力名称显示为“恢复对象查询”和“恢复点查询”。不显示内部方法名。

## 结果返回展示
标题为“请选择恢复时间点”。展示推荐候选、备选候选、风险和确认动作。

## 禁止展示
不得展示内部检索源、内部能力名、对象类标识、原始参数、原始返回、连接串、日志或堆栈。

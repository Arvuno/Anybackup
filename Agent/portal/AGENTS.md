# Agent 工作约定

## 目的

本项目的产品样式、UI 交互和视觉决策，默认优先对齐 V9 设计规范。

## 样式最高依据

凡是涉及产品外观、页面布局、交互细节或视觉表现的修改，首要参考以下文档：

- `D:\work\web_app_agent\docs\V9用户体验体系规范.md`

适用范围包括但不限于：

- 页面布局
- 侧边栏结构
- 登录页样式
- 字体与字号
- 间距
- 圆角
- 阴影
- 色彩使用
- 输入框、按钮、卡片样式
- AI 原生视觉表现
- hover / active / focus / disabled 状态

## 优先级规则

当多个样式依据发生冲突时，按下面顺序处理：

1. `docs/V9用户体验体系规范.md`
2. `docs/UI原型对齐规则-简版.md`
3. `src/` 中现有可复用组件与设计 token

如果当前实现与 V9 不一致，默认优先把实现往 V9 靠拢；除非用户明确要求保持其他口径。

## UI 修改工作规则

在修改任何 UI 相关代码前：

1. 先阅读 `docs/V9用户体验体系规范.md` 中对应章节
2. 尽量复用现有 token 与共享组件
3. 避免在业务组件里临时发明一次性视觉样式
4. 保持界面克制、现代、偏工作台风格

## 适用范围

V9 作为默认标准，适用于所有主要产品界面，尤其包括：

- 登录页
- 应用侧边栏
- 会话侧边栏
- 聊天区域
- 设置页
- 用户信息区域
- 弹窗与结构化富内容区块

## 默认判断原则

如果拿不准，优先选择更符合下面特征的方案：

- 更接近 V9
- 更轻
- 更紧凑
- 更一致
- 更容易复用
